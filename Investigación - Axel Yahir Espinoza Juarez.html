MÓDULO 01 // RECURSOS OPERATIVOS

¿Con qué se mueve un proyecto?

Antes de poner fecha a algo, todo proyecto necesita tres tipos de recursos: gente que lo haga, herramientas para hacerlo y dinero para sostenerlo. Da clic en cada tarjeta para abrir la definición y un ejemplo:

Definición

Es el conjunto de personas que forman parte de un proceso, cada una con un rol, un perfil (sus habilidades y experiencia) y responsabilidades específicas. Normalmente se define con un organigrama o con una matriz de responsabilidades, como la RACI, que deja claro quién hace cada cosa, quién decide y quién solo debe mantenerse informado.

> ejemplo

En un equipo que desarrolla una app, el Product Owner decide qué se va a construir, el Scrum Master organiza al equipo, los desarrolladores escriben el código y el Tester revisa que no haya errores antes de publicar. Si nadie tuviera un rol claro, dos personas harían lo mismo y otras tareas se quedarían sin hacer.

Definición

Son los bienes que se necesitan para que el proyecto opere: equipo de cómputo, servidores, software, licencias e insumos físicos. En un proyecto tecnológico esto no es solo "lo que se toca": la infraestructura digital también cuenta, porque sin ella el proyecto no puede correr.

> ejemplo

Una startup que está creando una app necesita laptops para el equipo, una licencia de Figma para diseñar las pantallas, un servidor en la nube (por ejemplo AWS) para alojar la aplicación, y herramientas como GitHub o Slack para organizar el trabajo. Si falta cualquiera de estos, el desarrollo se detiene.

Definición

Es el presupuesto necesario para ejecutar el proyecto. Un costo operativo es el gasto diario para mantenerlo funcionando (luz, internet, sueldos). Los costos fijos se mantienen igual sin importar cuánto avance el proyecto, como la renta de una oficina; los costos variables cambian según el uso o el avance, como las horas extra o el consumo de servidor. Proyectar estos costos desde el inicio es vital porque evita quedarse sin presupuesto a la mitad del camino.

> ejemplo

Una empresa de software calcula que pagará 50,000 pesos al mes en sueldos (costo fijo), más lo que gaste en servidores según cuántos usuarios tenga la app ese mes (costo variable). Si no proyecta esto desde el inicio, puede quedarse sin dinero antes de terminar de lanzar el producto.

MÓDULO 02 // PLANIFICACIÓN DEL TIEMPO

El Gráfico de Gantt

El Gráfico de Gantt es una herramienta visual que organiza todas las tareas de un proyecto sobre una línea de tiempo. Sirve para que cualquier persona del equipo, no solo el gerente, pueda ver de un vistazo qué se está haciendo, cuándo empieza, cuándo debe terminar y quién es responsable de cada cosa.

Usada para controlar proyectos porque es fácil de leer en segundos, permite detectar un retraso antes de que afecte a las demás tareas, y deja claro quién debe responder por cada actividad.

> por qué se usa tanto

Porque convierte un plan complicado (fechas, dependencias, responsables) en una sola imagen que se entiende sin necesidad de leer un reporte completo.
